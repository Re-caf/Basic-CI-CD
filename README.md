# Лабораторная работа: Настройка CI/CD пайплайна с GitLab Runner

## Суть задания

### Part 1. Настройка GitLab Runner
```markdown
1. Развернуть Ubuntu Server 22.04 LTS
2. Установить и зарегистрировать GitLab Runner:
   - Скачать официальный пакет
   - Зарегистрировать runner для проекта DO6_CICD
   - Использовать URL и токен из задания
3. Подготовить дамп ВМ после настройки

Part 2. Этап сборки
1. Реализовать этап CI для сборки:
   - Для SimpleBashUtils или code-samples из DO
   - Использовать Makefile проекта
2. Настроить артефакты:
   - Сохранять бинарные файлы
   - Установить срок хранения 30 дней
3. Конфигурация в .gitlab-ci.yml

Part 3. Тест кодстайла
1. Реализовать этап проверки стиля:
   - Запуск clang-format
   - Фейл пайплайна при ошибках
   - Отображение вывода утилиты
2. Требования:
   - Автоматический запуск
   - Блокировка при несоответствии стилю

Part 4. Интеграционные тесты
1. Написать этап тестирования:
   - Для SimpleBashUtils - существующие тесты
   - Для code-samples - новые тесты (bash/python/C)
2. Условия запуска:
   - Только после успешной сборки
   - Только при passed кодстайла
3. Отчетность:
   - Четкий вывод результата
   - Фейл пайплайна при ошибках

Part 5. Деплой на продакшн
1. Развернуть вторую ВМ Ubuntu 22.04 LTS
2. Реализовать CD этап:
   - Ручной запуск после успеха CI
   - Bash-скрипт для деплоя через ssh/scp
   - Копирование в /usr/local/bin
3. Требования:
   - Ясная логика скрипта
   - Обработка ошибок
   - Подготовка дампа ВМ
